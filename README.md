# margin-tracker（制度信用 週次チェック自動化）

JPXが毎週公表する「銘柄別信用取引週末残高」PDFを自動で取り込み、
ウォッチ中の銘柄について **制度信用の売残・買残・前週比・倍率** の一覧表を作ります。

- **制度信用倍率 = 制度買残 ÷ 制度売残**
- 毎週 **火曜18:30（日本時間）** に自動実行 → `report.md` が更新されます
- 過去分は `history.csv` に積み上がります（推移の分析に使えます）

---

## 出来上がったあとの使い方（これだけ）

毎週火曜の夜以降、リポジトリの **`report.md`** を開くだけです。
GitHubのファイル一覧から `report.md` をクリックすると、表がそのまま表示されます。

（自動実行に失敗していないかは、`report.md` 冒頭の「データ基準日」が
最新になっているかで確認できます）

---

## セットアップ手順（初回のみ・ブラウザだけで完結）

### 1. リポジトリを作る

1. GitHubの画面右上の **「＋」** → **「New repository」**
2. **Repository name** に `margin-tracker` と入力
3. **Public**（公開）のままでOK
   - 中身はJPXの公開データと銘柄コードだけなので問題ありません
   - Publicなら自動実行（Actions）が **無料・無制限** です
   - Privateにもできますが、無料枠に月2,000分の上限があります（この用途なら十分足ります）
4. 他は触らず、下の **「Create repository」** を押す

### 2. ファイルをアップロードする

1. 作成直後の画面にある **「uploading an existing file」** リンクを押す
   - 見当たらなければ **「Add file」** → **「Upload files」**
2. 配布されたフォルダの中身を **ドラッグ＆ドロップ**
   - **重要**: `margin-tracker` フォルダごとではなく、**中身のファイルを選んで**入れてください
   - `.github` フォルダも忘れずに入れてください（これが自動実行の本体です）
3. 下の **「Commit changes」**（緑ボタン）を押す

アップロードするもの:

```
margin_tracker.py
watchlist.txt
requirements.txt
README.md
.gitignore
.github/workflows/weekly.yml   ← これが無いと自動実行されません
```

### 3. 書き込み権限をONにする（重要）

これをしないと、結果をリポジトリに保存できずエラーになります。

1. リポジトリの **「Settings」** タブ
2. 左メニュー **「Actions」** → **「General」**
3. 一番下の **「Workflow permissions」** で
   **「Read and write permissions」** を選ぶ
4. **「Save」** を押す

### 4. 動作テスト（火曜まで待たずに今すぐ確認できます）

1. **「Actions」** タブを開く
2. 「I understand my workflows, go ahead and enable them」が出たら押す
3. 左の **「週次 信用残高チェック」** をクリック
4. 右側の **「Run workflow」** ボタン → 緑の **「Run workflow」**
5. 30秒ほど待ってページを再読み込み → 実行中の行をクリック

**緑のチェック✅** が付けば成功です。
リポジトリのトップに戻り、`report.md` が出来ていれば完了です。

以降は放っておいても毎週火曜に自動更新されます。

---

## 銘柄を追加・削除したいとき

1. `watchlist.txt` をクリック
2. 右上の **鉛筆マーク（Edit）** を押す
3. 銘柄コードを1行ずつ編集（`#` で始まる行はメモ扱いで無視されます）
4. **「Commit changes」** で保存

次回の自動実行から反映されます。すぐ反映したい場合は
「4. 動作テスト」と同じ手順で手動実行してください。

---

## うまくいかないときは

| 症状 | 対処 |
|---|---|
| Actionsが **赤い❌** になる | ログの `Permission denied` → **手順3**の権限設定を見直す |
| `1銘柄も抽出できませんでした` | JPXのPDF様式が変わった可能性。**下の注意**を参照 |
| 一部銘柄が「抽出できず」 | コードの打ち間違い、上場廃止、信用対象外のいずれか |
| 実行はされたがreport.mdが古い | JPXがまだ今週分を公表していない（公表は火曜16:30頃） |

### ⚠️ 2026年10月頃の注意

東証が公表様式を変更する予定です。
様式が変わるとPDFの読み取りに失敗する（＝Actionsが❌になる、または
「抽出できず」が多発する）可能性があります。

その場合は `margin_tracker.py` の **`ROW_RE`（行の読み取りルール）** の
修正が必要です。エラーが出たら、新しいPDFを添えて相談してください。

## 仕組み（参考）

```
JPXのページ → 最新PDFのリンクを探す → ダウンロード
  → pdfplumber で全85ページを解析
  → watchlist.txt の銘柄だけ抜き出す
  → report.md（今週の表）と history.csv（履歴）に書き出す
  → 結果をリポジトリに自動コミット
```

PDF1行の並びは以下の通りで、後ろから2番目が **制度買残**、
4番目が **制度売残** です（`▲` はマイナス）。

```
B 銘柄名 普通株式 コード ISIN
  売残合計 前週比 買残合計 前週比
  売残一般 前週比 売残制度 前週比
  買残一般 前週比 買残制度 前週比
```

安全のため、同じ基準日のデータは `history.csv` に**二重に追記されません**。
（保険の再実行が走っても数字が狂わないようにしてあります）
